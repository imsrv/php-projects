Help Section - Create / Edit subscribtion
`Main info` section:
Here you could specify Subscription name and admin e-mail for this subscription. You could use `Subscribtion name` variable in letters as #subscribtion_name# - this variable will be replaced to string you specify.

`Other settings` section:
Here you could set how many e-mail should be on each page when you manage this subscription. Default value for this variable you could set on `Settings screen` [here]
You could set expiration days for every subscription - it mean your subscribers will be placed in Expire section and will not get any e-mail. You could manually move such subscribers to `Active section`.

`Process users that send letters` section:
Here you could set information for pop3 account. This account will be checked every minute and all users that write letter to e-mail you specify will be added to subscription. Also in subscription will be added subscriber's full name is it set in `From` field. This e-mail will be deleted.

`Processing e-mails` section:
Here you could specify e-mail address. On this address will be send letters with user information that subscribe to this subsctiption and form fields

`Subscribe page` section:
This page will be showed your subscriber when subscribe via form. You could specify in this section subscribe text (html page) or url where will be redirected user after subscribe (for example subscribe.html)

`Unsubscribe page` section:
This page will be showed your subscriber when unsubscribe. You could specify in this section unsubscribe text (html page) or url where will be redirected user after unsubscribe (for example unsubscribe.html)

`Send e-mail when unsubscribe` section:
Here you could set letter text for unsubscribe e-mail. This e-mail will be send to user when it unsubscribe from your news.

`Use confirm URL`
Select this section if you need to user double opt-in feature. Users will receive confirmation e-mail with link (random link). It user clicks on this link - it become active users and will receive your e-mail.
Here you could specify text that user will view after approving or url - where this user will be redirected.

`Approve users after they subscribe` section:
Select this section, if you need to approve users that subscribed on your newsletters.
This users will receive e-mail (approve email) when you approve or not approve email if you delete user from subscription.
All users in this subscription will placed first on not approved section, and then (after approving) they moved to active users.

`Description` section:
This is information text only for admins.
