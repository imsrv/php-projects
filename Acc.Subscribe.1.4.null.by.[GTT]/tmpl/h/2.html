Help Section - Create / Edit newsletters
`Additional header fields` section:
Here you could specify such variables in header of letter as Bcc, or X-Mailer.

`HTML part` section:
You could upload html in this section from your hard drive, and your users will receive letters in html format.
In this html you could use any variables (description bellow)

`Attache path` section:
Your could upload files from your hard drive, from web or from server. This is a good feature to send dynamic created newsletters. For example, you could create letters, that will be send every monday and you have a script, that generate news. then you simply set path to this script on this section (upload from web) and your users will receive latest newsletters.

`Schedule` section:
Here you could set when this letter should be send to users.

Variables:
In your newsletters, you could use following variables, to make newsletters personalise:
#email#, #first_name#, #last_name#, #full_name#, #company#, #language#, #phone#, #icq#, #address#, #address2#, #city#, #state#, #country#, #birthdate#, #age#, #sex#, #website#, #heard#, #business#, #date_sub#, #date_unsub#, #ip#, #lettersformat#, #additional1#, #additional2#, #additional3#, #additional4#, #additional5#, #additional6#, #additional7#, #additional8#, #additional9#, #additional10#, #additional11#, #additional12# - user information.
#date#, #time#, #datetime# - date and variables.
#subscribiton_name#, #confirm-subscribe-url#, #unsubscribe_url#, #site-name#.
#rand=10# - will be replaced to random scring (10 character)
#file=/usr/home/some_file# - will be replaced to content of this file.
#url=www.somedomain.com# - will be replaced to content on this url.
You could also use :
#file=/usr/home/somedir/#email## (#email could be replaced to any different variable). or #url=www.google.com?q=#website##

To track how many e-mails was been read, please insert into html part #track_code# variable - this variable will be replaced to image with sizes 1x1. /Note: this feature available for letters that you send with `bulk sending` feature/
