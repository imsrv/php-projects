Special Tags
Throughout AvidNews software, there are special tags you may use to enhance your templates.
Some of these tags will only work in certain templates, so please take note, as to which templates they work with.

Tags for your News Templates
These tags only work under your "Template Management" section. Please read the description of the tags, to better understand which template they will work in, and what they do.

$news_info[id]
(only works in Headline Layout template)
This tag will display the ID number associated with with a particular news article. This should always be included in any link, when linking to a specific article

Example: http://www.yourdomain.com/news/viewarticle.php?id=$news_info[id]

The default template that comes with AvidNews, already has this laid out for you.

$news_info[headline]
(only works in Headline Layout template)
This tag will display the title of your news article, for each headline. Idealy this is used in your headline links

The default template that comes with AvidNews, already has this laid out for you.

$news_info[date]
(only works in Headline Layout template)
This tag will display the date that your news article was written/approved, for each headline. You can choose where the date appears in your headline layout, by placing this tag to your liking.

The default template that comes with AvidNews, already has this laid out for you.

$news_info[blurb]
(only works in Headline Layout template)
This tag will display the description of your news article, for each headline. This is usually placed directly below your headline title.

The default template that comes with AvidNews, already has this laid out for you.

$printpage
(works in Article Header, Show Article, and Article Footer templates)
This tag will display a link to a plain, printable version of your article.

The default template that comes with AvidNews, already has this laid out for you.

$emailpage
(works in Article Header, Show Article, and Article Footer templates)
This tag will display a link to a form, for users to email your article to their friends, etc.

The default template that comes with AvidNews, already has this laid out for you.

$article_info[headline]
(works in Article Header, Show Article, and Article Footer templates)
This tag will display the title of your article. This is generally used at the top of the page. It may also be used in your title tags, of your Article Header template.

The default template that comes with AvidNews, already has this laid out for you.

$article_info[blurb]
(works in Article Header, Show Article, and Article Footer templates)
This tag will display the description of your article. This is generally used below the title, as the first paragraph of your article. It may also be used in your "description" meta tag, in your Article Header template.

The default template that comes with AvidNews, already has this laid out for you.

$article_info[text]
(works in Article Header, Show Article, and Article Footer templates)
This tag will display the contents or body of your article.

The default template that comes with AvidNews, already has this laid out for you.

$article_info[image]
(works in Article Header, Show Article, and Article Footer templates)
If the writer of an article has uploaded an image with his/her article, this tag will display that image, wherever you place this tag, in your article.

The default template that comes with AvidNews, does not have this laid out for you.

$article_info[name]
(works in Show Article template)
This tag will display the author's name, and link to their profile (which opens in a new page)

The default template that comes with AvidNews, does not have this laid out for you.

$article_info[added_by]
(works in Show Article template)
This tag will display the author's name only.

The default template that comes with AvidNews, does not have this laid out for you.

<!-- startprint --> / <!-- stopprint -->
(works in Show Article template)
These tags are used to designate the portion of your article that you wish to be printable, on the print only version of your article. Anything placed between these tags will display on your print only version.

The default template that comes with AvidNews, already has this laid out for you.

Tags for your Profile Templates
These tags only work under your "Profile Templates" section. Please read the description of the tags, to better understand which template they will work in, and what they do.

$profile_info[name]
(works in Show profile template)
This tag will display the author's name only.

The default template that comes with AvidNews, has this laid out for you.

$profile_info[photo]
(works in Show profile template)
This tag will display the writer's personal photo (if they have provided one). If the writer has not provided a personal photo of themselves, a default No Photo image will be shown.

The default template that comes with AvidNews, has this laid out for you.

$profile_info[bio]
(works in Show profile template)
This tag will display the writer's personal bio (if they have provided one).

The default template that comes with AvidNews, has this laid out for you.

$profile_info[website]
(works in Show profile template)
If the writer has filled out their information, and included a website URL, this tag will display their website address, and a link, which will open in a new window.

The default template that comes with AvidNews, has this laid out for you.

Tags to use directly in your Articles
These tags only work under your "Article Management" section. Please read the description of the tags, to better understand which template they will work in, and what they do.

%newpage%
(works in Article input box)
If a writer has written or is writing a long article, they can break their article up into seperate pages with this tag. Simply put, whenever they want to start a new page, they only need to type this tag into their article. If a writer has chosen to use this tag, it will display "Back" and "Next" links at the bottom of each page. Keep in mind however, that this will also break your Print Only version of the article, up into different pages as well.
