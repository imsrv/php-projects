Global Variables Help

Site Variables:
Although information such as the the path to your root directory, and the path to sendmail is best obtained from your ISP there is an included script with ICS that can often determine most of this information for you. Navigate to http://www.yourdomain.com/ICScgi/GetSiteInfo.cgi to have ICS retrieve this information for you.
Path To Sendmail
You may have to ask your ISP for the path to sendmail on your web hosting server. The path shown by default is used on many servers, but this is by no means standard. In order for the e-mail notification features to work you must specify the correct path.
Your E-Mail Address:
 The e-mail address spider visits and warnings will be sent to. Please enter the address in the following format: somename@domain.com
Absolute Path To Your Root:
 This is the absolute path to your web space on this hosting server. You may have to ask your ISP what the absolute path is. Note that this is NOT the same as your URL. Alternatively you can use the included script called abspath.cgi to determine the absolute path and other info about your account.
Path To cgi:
 DO NOT ADD THE LEADING OR ENDING SLASH WHEN SPECIFYING A DIRECTORY NAME OR PATH. This is the path to the directory that will hold all ICS CGI scripts. Let's assume you must run all cgi programs from a cgi-bin directory and you chose to name the directory to hold ICS cgi's 'ICScgi', you would specify the path to the cgi directory as cgi-bin/ICScgi If you can run cgi's anywhere and created 'ICScgi' in your root then simply enter ICScgi Incorrect would be /cgi-bin/ICScgi/ or cgi-bin/ICScgi/ or /cgi-bin/ICScgi
The Directory To Hold Search Engine Directories:
 DO NOT ADD THE LEADING OR ENDING SLASH WHEN SPECIFYING A DIRECTORY NAME OR PATH. This is the directory that will hold all subdirectories of search engines and the human visitor directory. These will subsequently hold replicas of pages to be served according to visitor. You may specify any directory name you wish. It is a good idea not to use the default of 'SEdirect' because if people know your directory structure then they can see your cloaked pages. The name you specify here must match what you named this directory when you created it.
The Directory To Hold IP And Log Files:
 DO NOT ADD THE LEADING OR ENDING SLASH WHEN SPECIFYING A DIRECTORY NAME OR PATH. This is the directory that will hold the IP Address list, and the logs for spider visits and human visits, as well as the spider footprint log. The default name is IPDirect. You may choose any name you wish and/or path to a sub-directory. It is a good idea to change the name for security reasons. The name you specify here must match what you named this directory when you created it.
The Directory To Hold ICS Help Files:
 This is best left at the default name of ICShtml while doing the install, but can be changed to another name later if you wish to hide it once ICS is completely installed. The name you specify here must match what you named this directory when you created it.
The Name Of This Domain:
 The URL address of the site ICS is being istalled in and will operate from.
HTML File Extension Used At This site
 IF YOU ARE NOT UPGRADING FROM AN EARLIER VERSION OF ICS YOU MAY IGNORE THE HTML VARIABLE AND THE URI VARIABLE. This variable is included for backward compatability to older ICS versions. It is only used by sites that are determining the page name by URI. If you are new to ICS simply leave this set to html, and leave the URI variable to no. If you are upgrading from a previous version of ICS and are using SSI statements which do not have a "?" question mark in them, and do not have the pagename in the SSI statement, then you are using the URI version. You should then set the Using URI Variable to yes and set the html page extension variable to what is used at your site for pages served by ICS.
Enable Directory Mirroring
 Set Enable Directory Mirroring to Yes
to mirror your directory structure within the HumanVisitor and the search engine directories.

i.e. if the page name on the SSI statement (or URI if enabled) contains a directory as well as a pagename then ICS will look for that directory and page name within the HumanVisitor and each of the search engine directories.

Set Enable Directory Mirroring to No
to have ICS look for all cloaked pages in the root of the HumanVisitor and the search engine directories.

i.e. if the page name on the SSI statement (or URI if enabled) contains a directory as well as a pagename then ICS will ignore that directory and look for that page name within the root of HumanVisitor and each of the search engine directories.

If you are unsure of whether or not you should enable directory mirroring then begin by setting Enable Directory Mirroring to No since it is initially easier, you can always decide to change it to Yes and mirror your directory and subdirectory structure later.
