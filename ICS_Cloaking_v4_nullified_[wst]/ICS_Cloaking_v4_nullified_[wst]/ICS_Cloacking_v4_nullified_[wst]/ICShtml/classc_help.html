What Is A Reverse Class "c" Match?
Reverse Class "c"
An IP address is made up of four octets like so- 11.22.33.44, each position can have a value ranging from 0 to 255. The numbers 0 and 255 are never used for normal device IP addressing, so that leaves 1 to 254. A class "c" is a term used to refer to the first three octets of the IP address with the fourth octet being any number from 1 to 254.

So what is a reverse class "c" match? It is a phrase we coined to describe a match whereby a spider uses an IP address from a class "c" range and in the ICS spider db another address from that range exists for that spider.

For example; let's assume that the Inktomi spider Slurp visits and requests a page and it is using IP address 216.35.185.43 and in the ICS IP data base there is a statement defining an IP address for Inktomi's Slurp that looks like this:

/216.35.185.90/=Inktomi=/Slurp/&

Normally no IP match would occur because the two IP addresses are not the same. But taking advantage of the fact that spiders often use many IP addresses from the same class "c" we can take the address the spider used and drop the last octet so that we compare 216.35.185. against the ICS IP list. If another address is found that begins with 216.35.185. then a strong indication that this is a spider is established.

In the example used here the IP address used by Slurp 216.35.185.43 would be considered a match to the IP address 216.35.185.90 found in the ICS IP address db.

When this technique is combined with either a User Agent that must also match, or a strong spider footprint, it becomes very powerful in detecting spiders that might otherwise be missed.
