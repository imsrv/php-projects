System Requirements
To install SnippetMaster on your web server, you must meet following requirements:

  • UNIX / Linux Web Server (such as RedHat Linux, Cobalt RaQ III, etc)
  • PHP4 (register_globals must be on)
  • IE 5.5 sp1 or higher with javascript enabled - Click here to download
  • Ability to change file/folder permissions

Upgrading From A Previous Version
If you have an existing installation of SnippetMaster and want to upgrade to the latest release then you simply have to replace all the files EXCEPT for the config.inc.php file.

To "upgrade" your config.inc.php file:

    1. Open the new config.inc.php file in a text editor.
    2. Use the settings from your existing file to change the values in the new file.
    3. Specify a value for any new settings that were not in your old file.
    4. Save the new config.inc.php file and then copy it over your old config.inc.php file.

NOTE: You should ALWAYS make a backup copy of your old files before starting an upgrade. Then, if the upgrade fails for any reason you can simply copy back your old files.

Installation

== Install SnippetMaster Program Files ==

1. Using your FTP program, create a new directory on your website to hold the
SnippetMaster program files.

For example: /home/username/www/snippetmaster

2. Open the "includes/config.inc.php" file with a text editor and make any necessary adjustments to the default values for each setting. (The file is "self-documenting". This means that there is a description of each setting to help you understand what it does and how to adjust it.)

3. Upload all the SnippetMaster files and folders into the new directory.

NOTE: The .html and .php files MUST be transferred in ASCII format.
The .gif, files MUST be transferred in BINARY format.

4. Start the program to be sure it works. The URL you must enter into your browser is the value specified for $URL in the config.inc.php file.

== Configure Client Web Pages ==

1. Using your web editing tool of choice (Dreamweaver, Frontpage, Notepad), open the file that you want your client to be able to edit.

2. Insert the following tags before and after each snippet of code you want your client to be able to edit:

<snippet name="You can name the snippet here">

Existing HTML goes between the snippet tags ...

</snippet>

NOTE: ** YOU MUST PLACE THE SNIPPET TAGS ON THEIR OWN LINE. ***
If you place any text on the same line as a snippet tag, your text will disappear and SnippetMaster will not work!

3. Using your FTP program, upload the file to the server. (ASCII format)

4. Change file permissions of the file to at least 666. (File permissions can easily be set by using the CHMOD command feature within your FTP program.)

SnippetMaster will run with the same permissions as the web server software. Since most hosting companies run their web servers using the "nobody" user, SnippetMaster will not be able to modify your web pages unless you first change their file permissions. (Your web pages are owned by whatever user you use to log into FTP.)

3. Test the page to make sure everything is working correctly and that's all there is to it. Your page is is now ready to deliver to your client!

NOTE: The demo.html page contains a sample of how the SnippetMaster tags
should appear within a web page.

    End-User Instructions
    This program allows you to access your web pages to make online WYWIWYG (What You See Is What You Get) edits without having to go back to your web designer each time. You may change the text that appears in the WYSIWYG editing area however you wish, and then save the page when you are finished.

    Don't worry - you can't "break" anything. The special HTML programming code and page formatting inside your web page will NOT be disturbed.

     1. Go to the SnippetMaster login page on your website.

    2. Log in with your designated username and password. If you click the "Remember me" checkbox a cookie will be placed on your computer and then any time you go to the login page your username and password will be automatically supplied.

    3. The navigation area of the program allows you to move from one folder to another. If you want to open a different folder from the one currently displayed, just select a different folder from the folder drop-down menu and then click the [open] button.

    4. If you open a folder that has any Snippet-enabled files in it, those files will be displayed in a separate "files" drop-down menu.

     5. If the file you wish to edit is shown in the "files" drop-down menu, click on the file to highlight it.

    Note: If the file you wish to edit is buried within several folders, you will have to repeat step 3 until you are in the correct folder and your file is listed.

    6. Click the [open] button to open the file for editing.

    7. You should now see the WYSIWYG editor, containing the HTML code from your web page. You may make changes as you wish using the controls in the editor toolbar.

    8. During editing, you may click the [Preview] button to open a new window and see how your changes look. Click the "Close this preview window" button to return to the editing page.

    9. If you're finished editing, click the [Save] button and your changes will be instantly saved and made live.

     10. Repeat Steps 3 through 9 for any other web pages you wish to edit.

    Methods Of Using SnippetMaster
    There are basically two ways you can use SnippetMaster to manage your web page content.

    Method 1

    You simply embed the SnippetMaster <snippet> tags around the html code you want to make editable. This method works great if you don't normally need to make changes to the remaining html code (ie: A Dreamweaver template). If you needed to change the remaining html code, you must be sure to download the most current revision of the web page before working on it. Otherwise, you might end up overwriting any changes your client has made to the editable html code.

    Method 2 - For advanced users only

    This method is a bit trickier, and involves creating a seperate file that contains only the snippet of html code you want to make editable.

    1. Create a new file called snippet1.txt that contains the html code you want to be editable surrounded by SnippetMaster <snippet> tags.

    2. You must now "embed" this file into your web page by using a standard Server Side Include or PHP call. ie: <!--#include file="snippet1.txt"-->

    Your client can edit the html code in snippet1.txt using SnippetMaster whenever they choose, and you never need to worry about overwriting any changes they have made when you re-publish the web page template.

    Adding A New Language
    Adding additional languages to SnippetMaster (PRO) is very easy. If you create a new translation file, please be sure to send a copy to info@snippetmaster.com and I will add the file to the download area so that others may use it.

    At this time, only SnippetMater PRO has the ability to use language files. The text displayed with SnippetMaster LITE is "built-in" and can not be easily modified.

    1. Change into the languages folder.

    2. Copy an existing translation file and rename the new file.

    NOTE 1: Your new translation file MUST end with a .php extension: (ie: newlang.php)

    NOTE 2: The name you use for the new translation file will be displayed in the drop-down selection box on the login page. For example, if you call your new translation file "german.php" then the drop-down selection box on the login page will have an entry called "german".

    3. Edit the new translation file.

    NOTE: Your translation file MUST contain only valid PHP code. (ie: each variable must have open & closing quotes, etc.)

    4. Save the new translation file and upload it into the /languages folder.

    A default language can be specified by setting the "$DEFAULT_LANGUAGE" variable located in the config.inc.php file.
