Shop-Script 2.0 TROUBLESHOOTING

Установил магазин. Появляется только витрина магазина, а при нажатии на какие-либо ссылки абсолютно ничего не происходит. Что делать?

      В настройках PHP (файл php.ini) укажите register_globals = On

Установил магазин. Как попасть в администраторскую часть?
К разделу администратора имеет доступ только пользователь-администратор.

      Его логин и пароль вводятся при установке магазина. Если Вы забыли его логин, Вы можете посмотреть его в файле cfg/connect.inc.php (константа ADMIN_LOGIN). Пароль администратора хранится в базе данных.

Установил магазин. Зарегистрировал пользователя-администратора. Когда пытаюсь войти под его логином, ничего не происходит. Что делать?

      Проверьте - возможно, в Вашем магазин отключены cookies.

Не закачиваются фотографии товаров и категорий, не сохраняются настройки, дополнительные страницы, не работает импорт из Excel, синхронизация баз данных, экспорт пользователей. В чем проблема?

      Это может быть связано с атрибутами папок где лежат скрипты, products_pictures, cfg.
Универсальных настроек дать нельзя - все зависит от конфигурации сервера Вашего провайдера. Проконсультируйтесь в службе технической поддержки провайдера по вопросу "какие должны быть минимальные атрибуты директорий, в которых скрипты могли бы создавать файлы?".
Как показывает опыт, с атрибутами 766 все должно работать правильно.

Категории и товары сортируются в непонятном порядке

      По умолчанию в скрипте используется сортировка по алфавиту (средствами СУБД - с помощью запроса '... ORDER BY name').
Причина, по которой сортировка может не работать, может заключаться в том, что СУБД использует кодировку, отличную от той, что установлена в броузере при вносе наименований товаров и категорий.
Например, в броузере может быть установлена кодировка win1251, а в СУБД MySQL - koi8r. Т.е. происходит сортировка "win-названия", используя таблицу символов koi.
К примеру, в MySQL это можно исправить запросом (которые лучше поставить сразу после соединения с базой данных):
set CHARACTER SET cp1251_koi8
команда в PHP: db_query("set CHARACTER SET cp1251_koi8") or die (db_error());
Для разрешения данной проблемы для других СУБД обращайтесь в службу поддержки хостинг провайдера.

Почему скрипт показывает количество товаров 0 рядом с каждой из категорий, когда у меня товары в категориях есть?

      Количество товаров в категориях считается в режиме администратора.
Зайдите в раздел редактирования товаров и категорий и нажмите кнопку "Обновить значение 'количество товаров' для категорий".

Я не хочу иметь возможность добавления товаров в корзину - мне нужен обычный каталог. Как это сделать?

      Самый легкий путь решения этого - ставить всем товарам значение "остатка на складе" равным 0.
Кроме этого в шаблоне не используйте тег {SHOPPING_CART} - тогда не будет ссылки на оформление заказа.
Таким же образом можно убрать возможность регистрации в магазине (тэг {REGISTER/HOME}).
Удаление этих тэгов на быстродействие скрипта не повлияет.

Почему мои товары не показываются на витрине как специальные предложения?

      На витрине показываются только те товары, для которых включена галочка "показывать на витрине", и для которых закачана фотография (не "иконка" и не "popup-фотография"). Первыми показываются товары с наибольшим рейтингом (которые задаются пользователи).
